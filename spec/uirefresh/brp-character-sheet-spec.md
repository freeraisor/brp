# BRP Character Sheet — Layout Specification

Спецификация поведения и интеракций для редизайна листа персонажа BRP. HTML-мокап показывает визуал в **заблокированном режиме** (отображение). Режим разблокировки (редактирование через таблицу) реализуется отдельно.

---

## 1. Общая структура

Трёхколоночный лейаут с фиксированной боковой панелью идентичности слева, рабочей областью по центру и вертикальным рейлом табов справа.

| Зона | Ширина | Скролл | Назначение |
|---|---|---|---|
| Sidebar | 260px | независимый | неизменная идентичность и состояние персонажа |
| Workspace | flex | независимый | контент активного таба |
| Rail | 52px | нет | переключение вкладок |

**Скролл независимый:** HP и ресурсы всегда видны вне зависимости от скролла рабочей области. Sidebar скроллится только если контент превышает высоту окна (при нормальном наполнении — не скроллится).

---

## 2. Sidebar (левая колонка)

Сверху вниз:

1. **Аватар** — квадратный, aspect-ratio 1. При разблокированном режиме кликабельный для замены.
2. **Имя + подзаголовок** — подзаголовок формата `Profession · Age`.
3. **Status icons** — строка из 6 стандартных BRP-состояний.
4. **Hit Points** — крупный бар, красный акцент.
5. **Power Points** — крупный бар, синий акцент.
6. **Divider**.
7. **Fatigue / Sanity / Resource** — компактные бары (~4px высотой), приглушённые цвета.

### 2.1. Статус-иконки

Стандартный набор BRP:

| Состояние | Иконка | Источник (BRP RAW) |
|---|---|---|
| Bleeding | капля | wound effects |
| Stunned | звёзды вокруг головы | stun rules |
| Prone | лежащая фигура | positional |
| Grappled | цепь / захват | grappling rules |
| Unconscious | ZZZ | 0 HP и ниже |
| Dying | череп | major wound / 0 HP |

**Поведение:**
- Левый клик по иконке — включить/выключить состояние
- Правый клик — принудительно снять состояние
- Hover — tooltip с названием
- Активное состояние: красный цвет + подсветка фона
- Неактивное: приглушённый серый

---

## 3. Rail (правая колонка табов)

Порядок иконок (сверху вниз), сгруппирован по частоте использования в бою:

| # | Таб | Иконка (Lucide-подобная) |
|---|---|---|
| 1 | Skills | dice |
| 2 | Combat | crossed swords |
| 3 | Health | medical cross |
| 4 | Items | package |
| 5 | Character | user |
| 6 | Social | users |
| 7 | Story | open book |
| 8 | Effects | spark |
| 9 | Dev | gear |

**Поведение:**
- Клик — переключение таба (один активный за раз)
- Активный: мятный акцент, левая граница 3px, фон темнее
- Hover: tooltip слева от иконки с названием
- Без текстовых подписей

---

## 4. Workspace modes

Два режима, переключаются кнопкой lock/unlock (уже реализовано).

### Locked (текущий мокап)
- Красивый отображающий вид
- Интеракции только для геймплея: броски, чеки опыта, избранное, перетаскивание
- Редактирование значений невозможно

### Unlocked (реализовано отдельно)
- Появляется таблица редактирования
- Можно менять значения, добавлять/удалять навыки, править характеристики

## 5. Skills tab

### 5.1. Заголовок таба
`SKILLS` — заголовок отображающего стиля (Cinzel, 22px), мятный цвет. Справа индикатор режима (`🔒 Locked`).

### 5.2. Characteristic skills block

Отдельный блок над остальными скиллами. **Не является категорией**, не collapse'ится. 8 карточек в сетке 4×2:

| Характеристика | Производный навык | Формула |
|---|---|---|
| STR | Effort | STR × 5 |
| CON | Stamina | CON × 5 |
| INT | Idea | INT × 5 |
| SIZ | Damage mod | см. таблицу (не %) |
| POW | Luck | POW × 5 |
| DEX | Agility | DEX × 5 |
| CHA | Charm | CHA × 5 |
| EDU | Knowledge | EDU × 5 |

**На карточке:**
- Сверху: имя навыка (красный акцент) + значение (мятный)
- Снизу: `BASE · value` (например `STR · 11`)

**Клик по карточке:** бросок под процент (или бросок по Resistance Table для SIZ — уточняется отдельно).

### 5.3. Toolbar

Над категориями:

- **Search input** (flex) — живой фильтр по мере ввода
  - При наличии запроса: все категории с совпадениями автоматически раскрываются
  - Категории без совпадений скрываются целиком
  - Поиск по имени и специализации
- **Sort button** — циклически переключает режим:
  1. `A-Z` (по базе, затем по специализации)
  2. `Z-A`
  3. `%↓` (по убыванию процента)
  4. `%↑` (по возрастанию процента)
  - Режим применяется к скиллам внутри каждой категории отдельно
- **Add button** (иконка `+`) — открывает диалог создания скилла

### 5.4. Категории

Каждая категория — collapse-контейнер:

- **Header:** drag handle (появляется на hover), chevron, название (Cinzel, красный), счётчик скиллов
- **Body:** список скиллов
- **Клик по header** (кроме drag handle) — collapse/expand
- **Состояние collapse сохраняется** на уровне персонажа (актор.flags.brp-sheet.collapsed-categories)
- **Drag на категорию** — перестановка категорий между собой
- Категории определяются данными скилла, не отдельной сущностью

### 5.5. Skill row

Сетка: `[drag] [name · spec · star] [exp-check] [percent]`

**Имя + специализация:**
- Плоский список: `Pilot · Космический корабль` в одной строке
- База (`Pilot`) — основной текст
- Специализация (`Космический корабль`) — курсив, приглушённый
- Sort по имени: сначала база, потом специализация в алфавитном порядке

**Experience check:**
- Чекбокс, ставится при успехе для будущего роста навыка
- Клик по чекбоксу — toggle (без срабатывания броска)
- `event.stopPropagation()` обязателен

**Percent:**
- Пилюля в мятном стиле (граница + полупрозрачный фон)
- Минимальная ширина для выравнивания колонки

**Drag handle:**
- Невидим по умолчанию, появляется на hover ряда
- Cursor: grab
- Перетаскивание внутри категории и между категориями

### 5.6. Интеракции со скиллом

| Действие | Эффект |
|---|---|
| Hover | подсветка ряда |
| Left click | бросок под процент |
| Right click | контекстное меню |
| Click по exp-check | toggle чекбокса, без броска |
| Drag | перемещение |

### 5.7. Контекстное меню

Три пункта:

1. **Mark as favorite** / **Unmark favorite** — переключает избранное
2. **Duplicate** — создаёт копию скилла ниже в той же категории. Избранное на копию не переносится.
3. **Delete** — удаление, red-акцент

Меню закрывается по клику вне его или после выбора пункта. Позиционирование с учётом границ окна.

### 5.8. Избранное (favorite)

По требованию — **выделение, а не закрепление**. Избранный скилл остаётся в своей категории, но визуально подсвечен:

- Жёлто-янтарный градиент слева направо (fade в прозрачность)
- Левая граница 2px янтарного цвета
- Звёздочка ★ справа от названия

Логика сортировки НЕ меняется — избранные не всплывают наверх.

## 6. Drag & drop

В мокапе заложены DOM-хуки (`.cat-drag`, `.skill-drag`), но сама логика drag не реализована. Для Foundry VTT рекомендуется **SortableJS** (`Sortable.create()`) для обоих уровней:

1. **Skill внутри категории** — перестановка и перенос в другую категорию (`group: 'skills'`)
2. **Категория целиком** — перестановка категорий (`group: 'categories'`)

Drag handle: элемент с классом `.cat-drag` / `.skill-drag`, опция `handle` в Sortable.

После drop — сохранять порядок в данные актора (например, `item.sort` или custom flag).

## 8. Типографика

- **Display font:** Cinzel (500/600/700) — заголовки, названия навыков, лейблы ресурсов
- **Body font:** Inter (400/500/600) — всё остальное
- Размеры: заголовок таба 22px, имя персонажа 18px, метки 10–13px
- Letter-spacing на заголовках и лейблах — 1–4px для гримдарк-ощущения
- Все заголовки в uppercase через `text-transform`

---

## 9. Что НЕ входит в мокап

Требует отдельной реализации в Foundry:

- **Автосвязь оружия со скиллом** — `weapon.skill_id → skill.percent`. Уже реализовано в системе.
- **Расчёт процентов характеристик** — формулы × 5 применяются системой при изменении базы.
- **Диалог добавления скилла** — клик по кнопке `+` открывает форму, сейчас заглушка.
- **Drag & drop** — подключить SortableJS.
- **Сохранение состояния** (collapsed categories, favorites, порядок) — флаги актора.
- **Режим unlock** — реализован отдельно, переключение через замок.
- **Другие табы** (Combat, Health, Items, Character, Social, Story, Effects, Dev) — содержимое в этом мокапе не прорабатывалось.
